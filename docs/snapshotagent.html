Snapshot Agent

Send comments on this topic

← Previous Next →

A snapshot agent is a process that runs on an origin system and is responsible for providing snapshots of the origin applications data on demand.

 

The snapshot agent connects to the Kafka snapshot request topic as a consumer to the snapshot response topic as a publisher.

 

When a snapshot request is received the snapshot agent:

 

1.Unloads each of the configured data files to a portable format.

2.Compresses the exported data using LZ4 compression to a LZ4 archive in an outer TAR archive.

3.Uploads the TAR archive to network storage (usually an S3 bucket).

4.Sends a "snapshot complete" message to the snapshot response topic

5.Deletes the local copy of the TAR archive.

 

The name of the TAR archive includes the current offset in the main event topic so that the exact point in the time-line of data changes the snapshot data is associated with.

 

The Snapshot Agent for Windows and Linux is written in Rust. The Snapshot Agent for OpenVMS is written in VMS-C.
