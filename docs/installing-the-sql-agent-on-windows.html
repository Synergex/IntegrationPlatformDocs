Installing the SQL Agent on Windows

Send comments on this topic

← Previous Next →

<THIS TOPIC IS A WORK IN PROGRESS>

 

Installing the SQL Agent on a Windows target system involves downloading and running a product installation.

 

The installation can be downloaded from <to be determined>.

 

The name of the installation file will be as shown below, where n.n.n.n is the version number of the product you have downloaded.

 

Synergex-SIP-SQLAgent-n.n.n.n.msi

 

Executing the installation requires administrator privileges.

 

The product will be installed into the directory C:\Program Files\Synergex\SIP\SQLAgent

 

 

Upgrading the Product

 

To upgrade the SQL Agent to a new version of the product by:

 

•Review the release notes for the new version so that you are aware of any prerequisite tasks, or tasks that must be completed after the upgrade.

 

•Download the installer for the new version and copy it to the target system.

 

•Check that you still have the installer for the version you are currently running, in case you need to revert to that version for any reason. If you don't have it, you can download it from <to be determined>.

 

•Perform any prerequisite tasks as defined in the release notes.

 

•When you are ready to perform the upgrade, stop the SQL Agent service.

 

•Execute the installer to perform the upgrade.

 

•Perform any post upgrade tasks as defined in the release notes.

 

•Restart the SQL Agent service.

 

 

Removing the Product

 

Removing the DQL Agent from a Windows target system can be achieved by:

 

•Ensuring that the SQL Agent service is not running.

 

•Use either Installed Apps in the Windows Settings App or Programs and Features in the Windows Control panel to remove the product from the system. Look for the product name:
