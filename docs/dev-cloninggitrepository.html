Cloning the Git Repository

Send comments on this topic

← Previous Next →

 

You will need access to the following Git repository: https://github.com/Synergex/IntegrationPlatform. This is a PRIVATE repository, so before you can access it you will need to be specifically granted access to it. If you do not already have a GitHub user account then go create one now (the free tier is fine), and then request access by providing your GitHub username to Steve Ives or Jeff Greene.

 

IMPORTANT: The Git repository includes the use of Git submodules, which must be taken into account when cloning the repository. Please carefully follow the instructions below.

 

Configuring SSH Access to GitHub

 

If you would like to access GitHub via SSH instead of HTTPS then follow the instructions in Setting up SSH access to GitHub.

 

Location

 

We recommend cloning it the repository into a folder that does not include spaces in the path. We have tried to ensure that there are no hard-coded paths in checked-in files, but that is not entirely possible, so for maximum compatibility we recommend cloning the repository into C:\DEV\SYNERGEX\IntegrationPlatform.

 

Cloning the Repository

 

To clone using HTTPS:

 

git clone --recurse-submodules https://github.com/Synergex/IntegrationPlatform.git <location>

 

E.g.

 

git clone --recurse-submodules https://github.com/Synergex/IntegrationPlatform.git C:\DEV\SYNERGEX\IntegrationPlatform

 

To clone via SSH:

 

git clone --recurse-submodules git@github.com:Synergex/IntegrationPlatform.git <location>

 

E.g.

 

git clone --recurse-submodules git@github.com:Synergex/IntegrationPlatform.git C:\DEV\SYNERGEX\IntegrationPlatform

 

Fixing up the submodules

 

In theory, git submodules are great, but in practice they never seem to get cloned correctly and always require fixing up after cloning!
