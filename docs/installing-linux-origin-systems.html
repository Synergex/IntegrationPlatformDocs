Installation

 

Installing the Streaming Integration Platform on a Linux system involves downloading and running a product installation.

 

The installation can be downloaded from <to be determined>.

 

The name of the installation file will be as shown below, where n.n.n-n is the version number of the product you have downloaded.

 

sip-base-environment-n.n.n-n.el9.x86_64.rpm

 

The base environment includes the provision of two systemd services that, by default will run under a user account and user group named sip-service, which will be created during the installation if they do not already exist. If you prefer to have the environment owned by and the services run under a different user account or group, you can do so be creating the configuration file /etc/sysconfig/sip.conf and defining the user account and group to be used, like this:

 

# Customizes the service account used by Streaming Integration Platform services.

SVC_USER="username"

SVC_GROUP="groupname"

 

The installation is started using the package manager, and requires root privileges, like this:

 

sudo dnf install sip-base-environment-n.n.n-n.el9.x86_64.rpm

 

 

The product will be installed into the directory /usr/lib/synergex/sip/base-environment

 

Destination

 

The installation creates and/or places files in the following locations:

 

Location

Description

/usr/lib/synergex/sip/base-environment

Main base environment executables and other files.

/etc/synergex/sip

Configuration files.

/etc/iceoryx2

A configuration file used by one specific dependency library (iceoryx2.toml).

/etc/profile.d

Login scripts for interactive processes (sip-profile.sh and sip-profile.csh).

/etc/systemd/system

Service unit files (sip-message-broker.service and sip-snapshot-agent.service).

/var/log/synergex/sip

Various log files.

Service Management

 

The Streaming Integration Platform includes two applications that run as a standard Linux systemd services and can be controlled using the systemctl utility.

 

The names of the services are:

 

Application

Service Name

Snapshot Agent

sip-snapshot-agent

Message Broker

sip-message-broker

 

Some examples of common systemctl command are shown here:

 

To show the status of the service:

 

sudo systemctl status <service-name>

 

To restart the service:

 

sudo systemctl restart <service-name>

 

To stop the service:

 

sudo systemctl stop <service-name>

 

To unregister the service:

 

sudo systemctl disable <service-name>

 

To re-register the service:

 

sudo systemctl enable <service-name>

 

 

Upgrade

 

IMPORTANT: It is CRITICAL that Streaming Integration Platform version upgrades are only ever performed at a time when the origin application is completely inactive. Failing to do this will result in all replicated data sets becoming out of sync with the system of truth and will then require the reloading of those systems from a snapshot.

 

To upgrade the Streaming Integration Platform to a new version of the product:

 

•Review the release notes for the new version so that you are aware of any prerequisite tasks, or tasks that must be completed after the upgrade.

 

•Download the installer for the new version and copy it to the target system.

 

•Check that you still have the installer for the version you are currently running, in case you need to revert to that version for any reason.

 

•Perform any prerequisite tasks as defined in the release notes.

 

•When you are ready to perform the upgrade, stop the services.

 

sudo systemctl stop sip-snapshot-agent sip-message broker

 

•Verify that the services stopped:

 

sudo systemctl status sip-snapshot-agent

sudo systemctl stop sip-message broker

 

•Execute the installer for the new version. It is not necessary to remove the old version, the installer for the new version will perform an in-place upgrade:

 

sudo dnf install sip-base-environment-n.n.n-n.el10.x86_64.rpm

 

•Perform any post upgrade tasks as defined in the release notes.

 

•Restart the services:

 

sudo systemctl start sip-snapshot-agent sip-message-broker

 

 

Removal

 

Removing the Configuration Server from a Windows origin system can be achieved by:

 

•Ensuring the services are stopped.

 

sudo systemctl stop sip-snapshot-agent sip-message-broker

 

•Verify that the service stopped:

 

sudo systemctl status sip-snapshot-agent

sudo systemctl stop sip-message broker

 

•Similar to when the product was installed, removing the product is again achieved using the package manager:

 

sudo dnf remove sip-base-environment
